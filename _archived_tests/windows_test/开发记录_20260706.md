## 3. 根因分析

### 3.1 原版检测算法

```python
# vision_monitor.py 第 627-661 行
def _detect_person_from_depth(self, depth):
    fg_mask = (depth > 300) & (depth < 5000)  # 深度在 0.3~5m → 前景
    rows, cols = np.where(fg_mask)
    # → 包围盒 = 人体
```

### 3.2 失败原因

摄像头安装在 **20cm 高度、水平向前**。对于一个水平安装的低矮摄像头：

- 画面下半部分是地面
- 地面深度在 0.3m~5m 范围内（恰好对应 0.3~5000mm 阈值）
- **整片地面都被标记为前景**
- 人的像素混在地面中，无法形成独立包围盒

```
                   摄像头 (20cm高, 水平)
                   ┌───┐
                   │   │──────→ 光轴 (水平)
                   └───┘
                  ╱
                 ╱ 地面在画面下半部
    ────────────╱───── 深度 = 0.5~8m → 全在"前景"范围内！
```

### 3.3 为什么会"偶尔"检测到一次

只有人站得非常近（<1m）时，上半身超出地面范围，才形成独立包围盒。

## 4. 解决方案：地面感知检测算法

### 4.1 核心思路

不判断"离相机多远"，而是判断"离地面多高"：

```
逐像素离地高度 = 摄像头安装高度 − (行号 − 主点行) × 深度 / 垂直焦距
```

- **地面**: 离地高度 ≈ 0m → 排除
- **站立的人**: 离地高度 > 0.25m → 标记为人体候选

### 4.2 算法验证

| 场景 | 人体FG像素 | 地面识别率 | 检测结果 |
|------|-----------|-----------|---------|
| 纯地面 | 0 | 100% | ✓ 正确无人 |
| 1m处站立者 | 17,200 | 正常 | ✓ 检测到 |
| 2m处站立者 | 7,450 | 81.4% | ✓ 检测到 |
| 4m处站立者 | 1,875 | 94.6% | ✓ 可检测(边际) |

**结论**: 最佳检测距离 1-3m，4m 可检测但像素较少（24×74 px²）。

### 4.3 与板端代码的对应

| 原版代码 | 新版代码 | 说明 |
|---------|---------|------|
| `depth > 300 & depth < 5000` | `height_above_ground > 0.25` | 从"深度范围"改为"离地高度" |
| 无地面过滤 | 地面mask(离地<15cm) | 新增地面识别 |
| 纯numpy, <1ms | 纯numpy, <5ms | 性能几乎不变 |

## 6. 部署步骤

### 步骤1: 板端录制深度图

在 RDK X5 上：

```bash
# ── 终端1: 启动 MIPI 双目摄像头 ──
source /opt/tros/humble/setup.bash
renice -n -10 -p $$ 2>/dev/null
taskset -c 0,1 ros2 launch mipi_cam mipi_cam_dual_channel.launch.py

# ── 终端2: 启动 BPU 深度处理 ──
source /opt/tros/humble/setup.bash
taskset -c 2,3 ros2 launch hobot_stereonet stereonet_model.launch.py \
    stereo_image_topic:=/image_combine_raw \
    stereo_combine_mode:=1 need_rectify:=True alpha:=1

# ── 终端3: 录制深度图 ──
source /opt/tros/humble/setup.bash
unset RMW_FASTRTPS_USE_QOS_FROM_XML
unset FASTRTPS_DEFAULT_PROFILES_FILE
python3 save_depth_frames.py --count 30
# → 保存到 ~/depth_samples_XXXX/

# 拷贝到 PC (scp / U盘 / 网络共享)
```

### 步骤2: PC端分析调参

```bash
cd windows_test
pip install -r requirements.txt

# 交互调参模式（推荐）
python depth_analyzer.py --folder ~/depth_samples_XXXX/ --interactive --cam-height 0.20

# 拖动滑块调节:
#   - minHeightAboveGround: 离地高度阈值 (建议 0.20~0.30m)
#   - minFG Pixels: 最少前景像素 (建议 500~1000)
#   - depthMax: 最大检测深度 (建议 5~8m)
```


## 8. 火焰检测

火焰检测使用 HSV 颜色空间，PC端和板端算法参数完全一致，无需特殊适配：

```bash
# PC端测试
python fire_detector_standalone.py --camera 0
```

板端 `vision_monitor.py` 中 `detect_fire_color()` 函数使用相同的 HSV 范围。

## 9. 后续优化方向

1. **提高4m检测率**: 降低 `min_height_above_ground` 到 0.15m，或增加 RGB 辅助检测
2. **多人检测**: 当前包围盒方案只检测最大连通区域。可改为多轮廓检测
3. **降低深度噪声**: 增加时域中值滤波
4. **夜间检测**: 深度图不依赖光照，本算法天然支持。但火焰检测需要可见光
